# Changelog

## 2026-06-08

- FQDN обрабатывается как обывный текст, значение его не анализируется, в буквальном смвсле для matchera это **ANY**
- Изменена обработка FQDN-объектов в `core/resolver.py`.
- FQDN теперь учитываются при разрешении сетевых объектов и не блокируют правило, если DNS-резолвинг недоступен.
- FQDN-объекты интерпретируются как потенциально совпадающие, чтобы правило оставалось доступным при отсутствии DNS.
- В выводе FQDN отображаются как `FQDN:<имя>` или аналогичное текстовое представление, указывающее на FQDN-объект.
- Сделано описание изменений без включения исходного кода, только функциональные обновления.
